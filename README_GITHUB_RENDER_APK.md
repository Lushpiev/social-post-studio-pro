# APK без Windows: GitHub + Render

Этот вариант нужен, если вы хотите:
- загрузить проект в GitHub,
- развернуть сервер один раз,
- нажать запуск сборки,
- скачать готовый APK на телефон.

## Как это устроено
1. **Render** публикует сервер и веб-интерфейс по HTTPS.
2. **GitHub Actions** собирает Android APK-обёртку.
3. APK открывает уже размещённое приложение по адресу `APP_URL`.

## Что нужно сделать с телефона
### 1) Загрузить проект в GitHub
Залейте весь проект в новый репозиторий.

### 2) Развернуть сервер на Render
- В Render создайте новый **Blueprint** или **Web Service** из этого репозитория.
- Файл `render.yaml` уже добавлен.
- После деплоя получите адрес вида:
  `https://имя-приложения.onrender.com`

### 3) Добавить адрес приложения в GitHub
В репозитории GitHub:
- **Settings** → **Secrets and variables** → **Actions** → **Variables**
- создайте переменную:
  - Name: `APP_URL`
  - Value: `https://ваш-адрес.onrender.com`

### 4) Собрать APK
- Откройте вкладку **Actions**
- Выберите **Build Android APK**
- Нажмите **Run workflow**

### 5) Скачать APK
- После завершения откройте запуск workflow
- В блоке **Artifacts** скачайте `telegram-autopost-debug-apk`
- Внутри будет файл `app-debug.apk`

## После установки
Откройте APK, затем в приложении укажите:
- Telegram bot token
- chat_id канала
- при желании OpenAI API key

## Важное ограничение
Очередь и отложенные публикации работают на сервере. Поэтому сервер Render должен оставаться доступным.
